Разработан инструмент визуализации графа зависимостей для менеджера
пакетов. Готовые средства (менеджеры пакетов, библиотеки) для получения
зависимостей использовать не использовались.
1. Источником настраиваемых пользователем параметров являются опции
командной строки.
2. К настраиваемым параметрам относятся:
– Имя анализируемого пакета.
– URL-адрес репозитория или путь к файлу тестового репозитория.
– Режим работы с тестовым репозиторием.
– Имя сгенерированного файла с изображением графа.
– Подстрока для фильтрации пакетов.
3. При запуске приложения выводятся все параметры,
настраиваемые пользователем, в формате ключ-значение.
4. Реализована и продемонстрирована обработка ошибок для всех параметров.
5. Использован формат пакетов JavaScript (npm).
6. Извлечена информация о прямых зависимостях заданного пользователем пакета, используя URL-адрес репозитория.
7. Выведены на экран все прямые зависимости заданного пользователем пакета.
8. Получение графа зависимостей реализоватно алгоритмом DFS с рекурсией.
9. Не учитываются при анализе пакеты, имя которых содержит заданную пользователем подстроку.
10. Корректно обрабатываются случаи наличия циклических зависимостей.
11. Поддерживается режим тестирования. Вместо URL реального репозитория, есть возможность пользователю указать путь к файлу описания графа
репозитория, где пакеты называются большими латинскими буквами. Продемонстрирована функциональность этого этапа на различных случаях
работы с тестовым репозиторием.
12. Поддерживается режим вывода на экран порядка загрузки зависимостей для заданного пакета. Сравнить результаты с
реальным менеджером пакетов. Если есть расхождения в результатах, объяснить их наличие.
13. Продемонстрирована функциональность этого этапа на различных случаях работы с тестовым репозиторием.
14. Сформировано текстовое представление графа зависимостей на языке диаграмм Graphviz.
15. Сохраняется изображение графа в файле формата SVG.
16. Продемонстрированы примеры визуализации зависимостей для различных пакетов.
